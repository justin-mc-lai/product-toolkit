# 价值主张画布 (Value Proposition Canvas)

价值主张画布是 Alexander Osterwalder 提出的工具，用于设计和验证产品价值主张与客户需求的匹配度。

## 1. 价值主张画布概述

### 1.1 核心概念

```
┌─────────────────────────────────────────────────┐
│              价值主张画布                        │
├─────────────────────┬───────────────────────────┤
│    Customer Profile │      Value Map           │
│                     │                           │
│  ┌───────────────┐ │  ┌───────────────────┐  │
│  │     Jobs      │ │  │ Products/Services  │  │
│  │   (工作)      │ │  │   (产品/服务)      │  │
│  └───────────────┘ │  └───────────────────┘  │
│                     │                           │
│  ┌───────────────┐ │  ┌───────────────────┐  │
│  │    Pains      │ │  │  Pain Relievers   │  │
│  │    (痛点)     │ │  │   (止痛药)        │  │
│  └───────────────┘ │  └───────────────────┘  │
│                     │                           │
│  ┌───────────────┐ │  ┌───────────────────┐  │
│  │    Gains      │ │  │   Gain Creators   │  │
│  │    (收益)     │ │  │   (创造者)        │  │
│  └───────────────┘ │  └───────────────────┘  │
│                     │                           │
└─────────────────────┴───────────────────────────┘
```

### 1.2 与产品思考的关系

| 价值主张元素 | 对应产品思考 | 作用 |
|-------------|-------------|------|
| Jobs | Q1, Q3 | 理解用户工作场景 |
| Pains | Q4, Q8, Q17 | 识别用户痛点 |
| Gains | Q16, Q19 | 理解用户期望 |

## 2. Customer Profile (客户画像)

### 2.1 Jobs (工作)

用户想要完成的工作：

| 类型 | 描述 | 示例 |
|------|------|------|
| **功能性工作** | 用户需要完成的具体任务 | 编写报告、预订机票 |
| **情感性工作** | 用户想要感受的情绪 | 感到自信、感到安全 |
| **社会性工作** | 用户想要在社会中呈现的形象 | 被视为专家、获得认可 |

**模板**:
```markdown
### 功能性工作
- {工作1}
- {工作2}

### 情感性工作
- {工作1}
- {工作2}

### 社会性工作
- {工作1}
- {工作2}
```

### 2.2 Pains (痛点)

用户完成任务时遇到的负面情绪、风险和障碍：

| 类型 | 描述 | 示例 |
|------|------|------|
| **阻碍** | 使工作难以完成的因素 | 太耗时、太复杂 |
| **风险** | 可能导致的负面结果 | 数据丢失、金钱损失 |
| **负面情绪** | 用户感受到的负面情绪 | 沮丧、困惑、焦虑 |

**模板**:
```markdown
### 阻碍 (Obstacles)
- {痛点1}
- {痛点2}

### 风险 (Risks)
- {风险1}
- {风险2}

### 负面情绪 (Negative Emotions)
- {情绪1}
- {情绪2}
```

### 2.3 Gains (收益)

用户期望获得的结果和正面情绪：

| 类型 | 描述 | 示例 |
|------|------|------|
| **必要收益** | 必须获得的基本收益 | 功能正常、数据准确 |
| **期望收益** | 用户期望但非必须的 | 更快、更简单 |
| **超预期收益** | 用户未预期但会喜欢的 | 额外功能、惊喜体验 |

**模板**:
```markdown
### 必要收益 (Required Gains)
- {收益1}
- {收益2}

### 期望收益 (Expected Gains)
- {收益1}
- {收益2}

### 超预期收益 (Delighting Gains)
- {收益1}
- {收益2}
```

## 4. 价值匹配分析

### 4.1 匹配矩阵

```
              价值主张
          ┌─────────────────┐
    好   │   反向匹配     │  ✗ 不匹配
  的   │  (你提供过多)   │
  痛    └─────────────────┘
  点
  缓
  解    ✗ 不匹配      ✓ 匹配
  器   │ (你未解决)    │ (解决+匹配)
       └─────────────────┘
          ✗ 无痛点      ✓ 创造
           解          收益
```

| 匹配类型 | 描述 | 行动 |
|---------|------|------|
| ✓ 匹配 | 缓解答痛点 + 创造收益 | 保持 |
| ✓ 创造 | 无痛点但创造收益 | 保持，可作为亮点 |
| ✗ 不匹配 | 缓解答痛点但不匹配 | 调整 |
| ✗ 反向 | 你提供过多 | 简化 |

### 4.2 评分模板

```markdown
## 价值匹配评分

### 痛点-止痛药匹配
| 痛点 | 止痛药 | 匹配度 (1-5) |
|------|-------|--------------|
| {痛点} | {方案} | {评分} |

### 收益-创造者匹配
| 收益 | 创造者 | 匹配度 (1-5) |
|------|-------|--------------|
| {收益} | {方案} | {评分} |

### 总分: {总分}/10
```
