# 验收标准模板库

## 5维度基础模板（必选）

所有用户故事必须包含以下5个维度：

1. **正向流程** - 核心功能可正常使用
2. **边界校验** - 边界值/异常输入验证
3. **错误处理** - 异常场景友好提示
4. **成功反馈** - 操作后明确反馈
5. **性能** - 响应时间/并发要求

## 功能类型模板（可选）

| 类型 | 适用场景 | 核心验收点 |
|------|---------|-----------|
| CRUD | 增删改查基础功能 | 列表、新增、编辑、删除、详情 |
| 表单 | 用户输入、注册、编辑 | 必填校验、格式校验、提交成功 |
| 列表 | 列表展示、分页、排序 | 分页、排序、筛选、空状态 |
| 搜索 | 关键词搜索、筛选 | 搜索结果、空结果、特殊字符 |
| 业务流程 | 多步骤流程、状态流转 | 状态正确、流程完整、回退处理 |
| 实时推送 | WebSocket、消息通知 | 消息接收、离线处理 |
| 支付 | 交易、充值、退款 | 支付成功、支付失败、重复支付 |
| 上传下载 | 文件、图片、音视频 | 上传成功、进度显示、下载完成 |
| 权限 | 角色、权限、访问控制 | 无权限提示、权限验证 |
| 通知 | 消息、推送、邮件 | 发送成功、接收成功 |

## 边界值测试模板

```markdown
**边界校验**:
- [ ] 最小值-1 = 提示"超出范围"
- [ ] 最小值 = 正常处理
- [ ] 中间值 = 正常处理
- [ ] 最大值 = 正常处理
- [ ] 最大值+1 = 提示"超出范围"
```
