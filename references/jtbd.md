# JTBD (Jobs-to-be-Done) 用户任务理论

JTBD（Jobs-to-be-Done）是一种理解用户需求的框架，关注用户想要完成的任务和达成的目标，而非产品功能本身。

## 1. JTBD 概述

### 1.1 核心理念

```
用户不是在购买产品，而是在"雇佣"产品来完成一项工作
```

**关键洞察**:
- 用户购买的是进步，而非产品
- 痛点和解决方案之间存在"推动力"
- 功能 vs 成果：用户关心的是成果，不是功能

### 1.2 与产品思考的关系

| JTBD 阶段 | 输出 | 与产品思考关系 |
|-----------|------|--------------|
| Job Statement | 任务声明 | 对应 Q1-Q3 |
| Forces Analysis | 力量分析 | 对应 Q6-Q9 |
| Job Story | 任务故事 | 转化为用户故事 |
| Progress | 用户进步 | 对应 Q22 效果衡量 |

## 2. JTBD 核心概念

### 2.1 Job Statement (任务声明)

**结构**:
```
当我 [情境]，我想 [动机]，以便 [期望成果]。
```

**示例**:
```
当我每天早上通勤时，我想听新闻了解时事，以便在工作中能与同事交流时事话题。
```

### 2.2 Forces Analysis (力量分析)

```
         推动力 (Pushes)
            ↑
     ┌────┴────┐
     │  任务    │
     └────┬────┘
     ↓         ↓
 习惯    焦虑
(Habits) (Anxieties)
     ↓         ↓
     └────┬────┘
           ↓
      吸引力 (Pulls)
```

| 力量类型 | 描述 | 示例 |
|---------|------|------|
| **推动力 (Pushes)** | 用户当前状态的不满意 | 现有方案太慢、太贵、太复杂 |
| **吸引力 (Pulls)** | 新方案的吸引点 | 新方案更快、更便宜、更方便 |
| **习惯 (Habits)** | 用户固有的行为模式 | 习惯使用某个竞品 |
| **焦虑 (Anxieties)** | 用户对新方案的担忧 | 数据安全、学习成本 |

### 2.3 Job Story (任务故事)

**结构**:
```
当我 [情境]，我想 [动机]，以便 [期望成果]。
```

**与用户故事对比**:

| 用户故事 | JTBD Job Story |
|---------|---------------|
| 角色 + 动作 + 价值 | 情境 + 动机 + 成果 |
| "作为用户，我想..." | "当我...我想...以便..." |
| 功能导向 | 成果导向 |

## 3. JTBD 分析流程

### 3.1 第一步：发现任务

**问题**:
- 用户试图完成什么任务？
- 任务发生的情境是什么？
- 任务有哪些步骤？

**模板**:

```markdown
## 任务发现

### 核心任务
**任务描述**: {描述}
**发生情境**: {时间、地点、情绪}

### 任务分解
| 步骤 | 任务 | 当前解决方案 | 痛点 |
|------|------|-------------|------|
| 1 | {任务} | {方案} | {痛点} |
| 2 | {任务} | {方案} | {痛点} |
| 3 | {任务} | {方案} | {痛点} |
```

### 3.2 第二步：理解力量

**问题**:
- 什么推动用户寻求新方案？
- 什么吸引用户选择我们的方案？
- 用户有什么旧习惯？
- 用户担心什么？

**模板**:

```markdown
## 力量分析

### 推动力 (Pushes)
1. {当前痛点1}
2. {当前痛点2}
3. {当前痛点3}

### 吸引力 (Pulls)
1. {新方案吸引点1}
2. {新方案吸引点2}
3. {新方案吸引点3}

### 习惯 (Habits)
1. {旧习惯1}
2. {旧习惯2}

### 焦虑 (Anxieties)
1. {担忧1}
2. {担忧2}
```

### 3.3 第三步：定义成果

**问题**:
- 用户期望达成什么成果？
- 如何衡量成功？
- 有什么阻碍因素？

**模板**:

```markdown
## 期望成果

### 功能成果
- {用户想要完成的具体任务}

### 情感成果
- {用户想要感受的情绪}
- {用户想要避免的情绪}

### 社会成果
- {用户想要在社会中呈现的形象}

### 阻碍因素
- {阻碍用户达成成果的因素}

### 成果衡量标准
| 成果 | 衡量指标 | 当前方案 | 目标 |
|------|---------|---------|------|
| {成果} | {指标} | {当前值} | {目标值} |
```

## 5. JTBD 应用场景

### 5.1 产品定位

使用 JTBD 来:
- 理解用户真正的需求
- 找到产品差异化机会
- 定义产品价值主张

### 5.2 用户研究

使用 JTBD 来:
- 设计用户访谈问题
- 分析用户反馈
- 识别用户痛点

### 5.3 与其他方法论整合

```
JTBD → Design Thinking → 产品思考 → 用户故事 → 测试用例
```

| 阶段 | 输出 | 下一阶段输入 |
|------|------|------------|
| JTBD | Job Story | Design Thinking (Empathize) |
| Design Thinking | Persona, 痛点 | 产品思考 (Q1-Q5) |
| 产品思考 | 验收标准 | 用户故事 |
| 用户故事 | AC | 测试用例 |

## 6. 子命令

使用以下子命令调用 JTBD:

```
/product-toolkit:jtbd {功能描述}
```

**示例**:
```
/product-toolkit:jtbd 外卖订餐
/product-toolkit:jtbd 在线教育学习
/product-toolkit:jtbd 社区点赞功能
```
