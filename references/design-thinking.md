# Design Thinking 设计思维

Design Thinking（设计思维）是一种以用户为中心的创新方法论，通过同理心、定义、创意、原型、测试五个阶段解决复杂问题。

## 1. Design Thinking 概述

### 1.1 五阶段模型

```
     ┌─────────────┐
     │  Empathize  │  同理心 - 理解用户
     │   (同理心)   │
     └──────┬──────┘
            ↓
     ┌─────────────┐
     │   Define    │  定义 - 明确问题
     │   (定义)    │
     └──────┬──────┘
            ↓
     ┌─────────────┐
     │  Ideation   │  创意 - 产生想法
     │   (创意)    │
     └──────┬──────┘
            ↓
     ┌─────────────┐
     │  Prototype  │  原型 - 快速验证
     │   (原型)    │
     └──────┬──────┘
            ↓
     ┌─────────────┐
     │    Test     │  测试 - 验证假设
     │   (测试)    │
     └─────────────┘
```

### 1.2 与产品思考的关系

| Design Thinking 阶段 | 问题 | 输出 | 与产品思考关系 |
|---------------------|------|------|---------------|
| Empathize | 目标用户核心痛点是什么？ | Persona, 痛点列表 | 对应第0轮（可选） |
| Define | 真正的问题是什么？ | Problem Statement | 对应Q1-Q5澄清 |
| Ideation | 可能的解决方案？ | 创意清单, MoSCoW | 对应Q6-Q9假设 |
| Prototype | 如何快速验证？ | 原型方案 | 独立阶段 |
| Test | 如何验证假设？ | 测试反馈 | 对应Q21-Q24影响 |

## 2. 各阶段详细说明

### 2.1 同理心 (Empathize)

**目标**: 深入理解用户的需求、痛点、行为和动机

**方法**:
- 用户访谈
- 观察用户行为
- 情景模拟
- 用户旅程地图

**输出模板**:

```markdown
## 同理心阶段输出

### 用户画像 (Persona)
- **姓名**: {姓名}
- **年龄**: {年龄}
- **职业**: {职业}
- **目标**: {用户目标}
- **痛点**: {用户痛点}
- **行为**: {用户行为模式}

### 用户旅程地图
| 阶段 | 行为 | 触点 | 情绪 | 痛点 |
|------|------|------|------|------|
| 认知 | {行为} | {触点} | {情绪} | {痛点} |
| 考虑 | {行为} | {触点} | {情绪} | {痛点} |
| 购买 | {行为} | {触点} | {情绪} | {痛点} |
| 使用 | {行为} | {触点} | {情绪} | {痛点} |
| 推荐 | {行为} | {触点} | {情绪} | {痛点} |

### 痛点列表
1. {痛点1}
2. {痛点2}
3. {痛点3}
```

### 2.2 定义 (Define)

**目标**: 从同理心阶段收集的信息中提炼出核心问题

**方法**:
- 问题陈述 (Problem Statement)
- 5 Whys 分析
- 用户洞察总结

**输出模板**:

```markdown
## 定义阶段输出

### 问题陈述 (Problem Statement)
**用户痛点**: {描述用户面临的挑战}
**影响**: {这个问题如何影响用户}
**洞察**: {从同理心阶段获得的洞察}

### 机会声明
**我们想要**: {设计方向}
**目标用户**: {目标用户群体}
**预期结果**: {期望达成的效果}
**衡量标准**: {如何衡量成功}

### 约束条件
- {约束1}
- {约束2}
```

### 2.3 创意 (Ideation)

**目标**: 产生大量创意想法，找到解决问题的最佳方案

**方法**:
- 头脑风暴
- 逆向思维
- SCAMPER 法
- 六顶思考帽

**输出模板**:

```markdown
## 创意阶段输出

### 创意清单
| # | 创意描述 | 可行性 | 影响力 | 优先级 |
|---|---------|--------|--------|--------|
| 1 | {创意} | {高/中/低} | {高/中/低} | {高/中/低} |
| 2 | {创意} | {高/中/低} | {高/中/低} | {高/中/低} |

### MoSCoW 优先级
- **Must have**: {必须实现}
- **Should have**: {应该实现}
- **Could have**: {可以实现}
- **Won't have**: {本期不做}

### 选定方案
**推荐方案**: {方案描述}
**理由**: {选择理由}
**风险**: {潜在风险}
```

### 2.4 原型 (Prototype)

**目标**: 用最低成本快速构建产品原型，验证创意假设

**方法**:
- 纸质原型
- 低保真原型
- 高保真原型
- 交互原型

**输出模板**:

```markdown
## 原型阶段输出

### 原型类型
- [ ] 纸质原型
- [ ] 低保真 (Figma/草图)
- [ ] 高保真 (Figma/设计稿)
- [ ] 交互原型 (ProtoPie/Figma)

### 原型范围
**页面**: {页面列表}
**功能**: {功能列表}
**验证目标**: {需要验证的假设}

### 验证计划
| 假设 | 验证方法 | 成功标准 | 48小时内完成? |
|------|---------|---------|--------------|
| {假设} | {方法} | {标准} | {是/否} |

### 资源需求
- **时间**: {小时}
- **人员**: {角色}
- **工具**: {工具列表}
```

### 2.5 测试 (Test)

**目标**: 通过用户反馈验证原型，迭代改进

**方法**:
- 可用性测试
- A/B 测试
- 焦点小组
- 问卷调查

**输出模板**:

```markdown
## 测试阶段输出

### 测试计划
**测试对象**: {原型/功能}
**测试用户**: {用户类型}
**测试方法**: {方法}
**样本量**: {数量}

### 测试结果
| 假设验证 | 结果 | 反馈摘要 | 改进建议 |
|---------|------|---------|---------|
| {假设} | {通过/失败} | {反馈} | {建议} |

### 迭代计划
**需要迭代**: {是/否}
**迭代内容**:
1. {迭代项1}
2. {迭代项2}

### 结论
**产品-市场匹配**: {是/否/待定}
**下一步**: {下一步行动}
```

## 3. 使用场景

### 3.1 何时使用 Design Thinking

| 场景 | 推荐阶段 | 输出 |
|------|---------|------|
| 新产品探索 | 全部5阶段 | 完整产品概念 |
| 功能创新 | Empathize → Ideation | 功能创意清单 |
| 用户体验优化 | Empathize → Test | 改进方案 |
| 问题诊断 | Empathize → Define | 问题根因 |

### 3.2 与产品工具包整合

```markdown
## 产品工具包工作流

### 选项 A: 从 Design Thinking 开始
/product-toolkit:design {功能}
    ↓
生成 Persona + 痛点
    ↓
/product-toolkit:think {功能}
    ↓
5轮产品思考 (Q1-Q24)
    ↓
/product-toolkit:user-story
    ↓
生成版本化用户故事

### 选项 B: 从产品思考开始
/product-toolkit:think {功能}
    ↓
5轮产品思考
    ↓
(可选) /product-toolkit:design {功能}
    ↓
补充 Design Thinking 输出
```

## 4. 子命令

使用以下子命令调用 Design Thinking:

```
/product-toolkit:design {功能描述}
```

**示例**:
```
/product-toolkit:design 电商支付功能
/product-toolkit:design 新用户注册流程
```
