## 测试用例编写规范

### 1. 用例编号规范

```
{模块英文缩写}-{序号}
示例：
- LOGIN-001 (登录模块)
- ORDER-001 (订单模块)
- PAY-001 (支付模块)
```

### 2. 优先级定义

| 优先级 | 定义 | 覆盖率要求 |
|--------|------|-----------|
| P0 | 核心功能，阻断发布 | 100% |
| P1 | 重要功能，影响体验 | 80% |
| P2 | 次要功能，可接受缺陷 | 50% |

### 3. 测试步骤规范

每个步骤应该：
- 简洁明确
- 可执行
- 可验证

### 4. 预期结果规范

- 明确具体
- 可观测
- 可量化
