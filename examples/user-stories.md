# 用户故事示例

## 完整示例：电商平台商品收藏功能

### US-001: 商品收藏功能

**用户故事**: 作为电商平台的用户，我希望能够收藏喜欢的商品，以便下次快速找到并购买。

**优先级**: Should
**前置条件**: 用户已登录

**验收标准**:
- [ ] **正向流程**: 点击商品详情页的收藏按钮，收藏成功，按钮状态变为已收藏
- [ ] **边界校验**: 未登录用户点击收藏按钮，提示登录；已收藏商品再次点击取消收藏
- [ ] **错误处理**: 网络异常时收藏失败，提示"收藏失败，请稍后重试"
- [ ] **成功反馈**: 收藏成功后 toast 提示"收藏成功"，我的收藏页面数量+1
- [ ] **性能**: 收藏操作响应 < 500ms

---

### US-002: 查看收藏列表

**用户故事**: 作为电商平台的用户，我希望能够查看我的收藏列表，方便管理已收藏的商品。

**优先级**: Should
**前置条件**: 用户已登录

**验收标准**:
- [ ] **正向流程**: 进入我的收藏页面，展示所有已收藏商品列表
- [ ] **边界校验**: 收藏为空时显示空状态"暂无收藏，快去逛逛吧"
- [ ] **错误处理**: 加载失败显示重试按钮
- [ ] **成功反馈**: 显示商品图片、名称、价格、优惠信息
- [ ] **性能**: 列表加载 < 2秒，支持分页加载

---

### US-003: 批量管理收藏

**用户故事**: 作为电商平台的用户，我希望能够批量删除不需要的收藏商品，以便整理收藏列表。

**优先级**: Could
**前置条件**: 用户已登录且有多个收藏商品

**验收标准**:
- [ ] **正向流程**: 进入编辑模式，选择多个商品，一键删除
- [ ] **边界校验**: 全选/取消全选功能正常
- [ ] **错误处理**: 删除失败时提示"删除失败，请重试"，保留选中状态
- [ ] **成功反馈**: 删除成功后 toast 提示"删除成功"
- [ ] **性能**: 批量删除 < 1秒

## 用户画像示例

### Persona: 都市白领小美

**基本信息**:
- 年龄: 28岁
- 职业: 互联网公司产品经理
- 收入: 月入2万+
- 学历: 本科

**用户特征**:
- 追求品质生活，愿意为好产品付费
- 时间碎片化，喜欢快速决策
- 重度手机用户，每天2-3小时
- 注重隐私，不喜欢过多个人信息暴露

**用户旅程**:
```
发现商品 → 收藏比价 → 等待优惠 → 下单购买 → 分享推荐
```

## 产品思考示例

### 场景：用户说"我要做一个社区论坛的点赞功能"

**苏格拉底式追问过程**:

1. **澄清问题**
   - "用户为什么要点赞？是为了表达认可还是获取关注？"
   - "如果不点赞，用户现在用什么方式表达认可？"

2. **探索假设**
   - "我们假设用户需要点赞功能"
   - "我们假设点赞能提升用户活跃度"

3. **追溯原因**
   - "为什么认为点赞比评论门槛更低？"
   - "为什么不做其他互动功能？"

4. **评估影响**
   - "不做点赞功能，用户会流失吗？"
   - "做了点赞，对社区氛围有什么影响？"

**网状思维发散**:

- **用户维度**: 内容创作者vs内容消费者，活跃用户vs潜水用户
- **场景维度**: 刷动态时、看完内容后、分享前
- **业务维度**: 竞品做法、差异化、商业变现
- **体验维度**: 点赞反馈、动画效果、数字显示
- **风险维度**: 刷赞、舆情、攀比心理

## 功能拆解示例

### 需求：做一个在线考试系统

**一级拆解（Epic）**:
1. 用户模块
2. 题库模块
3. 考试模块
4. 成绩模块

**二级拆解（Feature）**:

#### 用户模块
- 注册/登录
- 个人信息管理
- 学习记录

#### 题库模块
- 题目管理（单选/多选/判断/问答）
- 题目导入
- 题目检索

#### 考试模块
- 创建试卷
- 设置考试规则
- 在线答题
- 自动计时
- 交卷功能

#### 成绩模块
- 自动判卷
- 成绩查询
- 成绩分析

**拆解成用户故事**:

- US-001: 作为考生，我能够注册账号以便使用系统
- US-002: 作为考生，我能够创建试卷以便组织考试
- ...
