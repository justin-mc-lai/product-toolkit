# 用户故事示例

## 完整示例：比赛创建功能

### US-001: 创建随机双打比赛

**用户故事**:
> 作为**羽毛球爱好者**，我希望**创建随机双打比赛**，以便**快速组织比赛，不必手动匹配队友和对手**。

**优先级**: Must
**前置条件**: 用户已登录
**所属场景**: 比赛管理

**验收标准**:
- [ ] **正向流程**: 可成功创建随机双打比赛，流程完整
- [ ] **边界校验**:
  - [ ] 4人比赛可创建成功
  - [ ] 9人比赛可创建成功
  - [ ] 10人+随机双打提示"超出人数限制"
  - [ ] 5人+混双提示"人数必须为偶数"
  - [ ] 3人+单淘汰提示"需补齐到4人"
- [ ] **错误处理**: 网络超时提示"创建失败，请重试"
- [ ] **成功反馈**: 创建成功后自动跳转详情页
- [ ] **性能**: 创建响应 < 2秒

**依赖**: 用户登录、比赛模块基础架构
**技术标签**: 前端小程序、后端API、数据库

### US-002: 邀请好友参加比赛

**用户故事**:
> 作为**比赛创建者**，我希望**邀请微信好友参加比赛**，以便**快速凑齐比赛人数**。

**优先级**: Should
**前置条件**: 比赛已创建
**所属场景**: 比赛管理

**验收标准**:
- [ ] **正向流程**: 可通过微信分享邀请好友
- [ ] **边界校验**:
  - [ ] 比赛人数已满时提示"人数已满"
  - [ ] 比赛已过期时提示"比赛已结束"
- [ ] **错误处理**: 分享失败提示"请重试"
- [ ] **成功反馈**: 好友成功接受邀请
- [ ] **性能**: 邀请响应 < 1秒

**依赖**: US-001、微信小程序分享API
**技术标签**: 微信分享、小程序

### US-003: 查看比赛排名

**用户故事**:
> 作为**参赛选手**，我希望**查看比赛排名**，以便**了解自己的比赛成绩**。

**优先级**: Could
**前置条件**: 比赛已结束
**所属场景**: 比赛管理

**验收标准**:
- [ ] **正向流程**: 可查看所有参赛选手排名
- [ ] **边界校验**:
  - [ ] 比赛进行中不显示排名
  - [ ] 无参赛记录显示为空
- [ ] **错误处理**: 数据加载失败显示占位符
- [ ] **成功反馈**: 排名数据展示清晰
- [ ] **性能**: 排名加载 < 3秒

**依赖**: US-001、比赛结果计算
**技术标签**: 数据展示
